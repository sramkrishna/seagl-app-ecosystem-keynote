Who and what is the App Ecosystem?

Let's first define, what I'm talking about - The Linux App Ecosystem. It's essentiallye everyone who is volunteers their time to support writing applications on the Linux platforms. Documentation, bug triagers, social engagers, toolkit writers, middleware, frameworks, designers - the Linux App Ecosystem is a meta ecosystem that involves every skill set you can think of. There is an important cross section that is also part of the app community
That's right - the maintainers, the people who write the apps, who help design them - even the people who just make you feel good just talking to them!

Tale of Two File Browsers

Had a file browser called konqueror - it allowed you to load any content from pdf and load it within the browser

for HTML content - they wrote a standards compliant HTML rendering image called KHTML

Small, efficient - built by the KDE Community

Had a file browser called GNOME Midnight Commander (gmc) - not the most sophisticated of file browsers.

A guy named Andy Hertfield built a start up called 'Eazel', and started writing a GNOME based file browser called nautilus.

The idea was to add an internet attached storage - essentially they were working on DropBox before Dropbox."

The startup eventually failed - and most everyone went back to Apple.

A gal named Lisa Melton (deadname: Don Melton) started a project along with many of the former members of Eazel. Apple was interested in doing their own browser.

KHTML became the perfect choice for them because it was light, fast, and standards compliant and a perfect base for their own project

That project ended up being called 'WebKit'.

So essentally the work of two projects ended up creating 'WebKit' which now powers - today's web apps, and a plethora of browsers like Safari, as as former projects like Chrome. GNOME and KDE continue to use WebKit for their the web browser.

A Renaissance is Happening

The genesis of this renaissance comes from the effects of improved tooling and software engineering.

Everyone moves to Git.

Flatpak is introduced. Solving a lot of problems with packaging.

Flatpak - Solving the packaging problem

Every distro had their own toolchain of various versions.

Every distro would have their own bug database for app ecosystem software.

If a closed source app To reach the greatest audience you'd have to package both debian and rpm.

GNOME moves to GitLab. A number of other projects follow suit - KDE, Debian, Fedora, and so on

This is the turning point for the entire ecosystem because we have now fully moved to modern tools with capabilities like CI/CD, Issues, Wikis.

GNOME OS arrives...

GNOME OS - an idea whose time has come!

A dream come true - literally. GNOME has been talking about GNOME OS for a decade. The ability to drive GNOME's vision on their terms.

GNOME OS allows:

Designers to prototype new ideas quickly.

Provide a distributable version of GNOME to be used in user studies.

Build robust Q/A pipelines to test software and new design patterns and/or regressions.

Allow the community to test run GNOME daily.

A highly reliable, faster paced development cycle!

Moved to a more modern display server. Wayland is a new display server that replaces X11 server.

Much reduced overhead, and focused on a drawing model rather an comprehensive model that included everything.

Finally, we're raising funds and making investments in accessibility, better safety via encrypted per user home directories.

As an ecosystem, we are now also focused on each other. KDE and GNOME, working together to build apps. Through effots like the Linux App Summit.

Let's focus on:"

Cooperating on standards.

Share ideas on design.

Provide a united front when working with other ecosystems - by an individual project, we might be small. But together we are powerful.

We have a lot of challenges still. Going back to why the question on why the desktop has not yet been successful.

We have the following issues.

We are not yet sustainable. A platfrom can only be sustainable if people can make a living off of it.

Desktop branding continues to be an issue where Distros and Companies want to apply their own branding with color schemes.

We have some tensions with packagers. We are the only platform where distribution isn't controlled by volunteers not associated with the project.

We've made great strides even withou the attention and investment that we used to have. We are only getting better. The app ecosystem is a small group, barely over 2000 core people. Our work speaks for itself.
Today on the FlatHub App store we have over 2343 desktop apps, serving over 88.3 TB of data per day. Representing over a 1.3 billion downloads this year. Average download of apps per month is 4 million. This year alone we added 500 new applications. The next steps now is to start being able to allow developers to handle payments, offer subscriptions, and so on. It will be an all new phase in Linux land.

These numbers will improve as we make more investments in building better documentation, improved infrastructure, and streamlined maintainability of code.

Also realize that investment in essentially the consumer part of the ecosystem means that other projects like the kernel, laptop companies all become more viable because consumptions means that more consumer hardware vendor will be willing to support Linux.
